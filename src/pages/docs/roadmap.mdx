# Roadmap

A look at what's coming next. These are the key milestones we're working toward to make Zaidan the definitive component toolkit for <SolidJS class="inline-block size-3.5 align-text-bottom mx-0.5" /> SolidJS.

<Steps>

### Zaidan Agent for Any SolidJS Codebase

<span class="inline-flex items-center rounded-full border border-sky-500/30 bg-sky-500/10 px-2.5 py-0.5 text-xs font-medium text-sky-600 dark:text-sky-400">In Progress</span>

Make the Zaidan agent available beyond this project. Any SolidJS codebase will be able to leverage Zaidan agent — from scaffolding new components to syncing registry updates — directly from their own project.

### Starter Templates

<span class="inline-flex items-center rounded-full border border-amber-500/30 bg-amber-500/10 px-2.5 py-0.5 text-xs font-medium text-amber-600 dark:text-amber-400">Planned</span>

Create ready-to-use starter templates to jumpstart building apps powered by the Zaidan registry. Pre-configured with our favorite stack — so you can skip the setup and start shipping.

### Sync Base UI Components

<span class="inline-flex items-center rounded-full border border-amber-500/30 bg-amber-500/10 px-2.5 py-0.5 text-xs font-medium text-amber-600 dark:text-amber-400">Planned</span>

Port the <a class="font-medium underline-offset-4 underline" href="https://base-ui-docs-solid.vercel.app/" target="_blank">Base UI (SolidJs port)</a> version of each component and block into the registry. This will provide an alternative primitive layer alongside Kobalte, giving developers more choices for accessible foundations.

</Steps>
